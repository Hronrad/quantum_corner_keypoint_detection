## 5. Data found in the uploaded zip

The uploaded `data.zip` contains six subdatasets after extraction:

| dataset | images | patches | patch size / image shape notes | purpose |
|---|---:|---:|---|---|
| smoke_readme_pipeline | 3 | 288 | 64x64 images, 9x9 patches | fastest pipeline sanity check |
| synthetic_keypoints_smoke | 24 | 2304 | 128x128 images, 21x21 patches | smoke QNN/baseline comparison |
| synthetic_keypoints_square_smoke | 8 | 768 | 80x80 images, 21x21 patches | tiny square-image check |
| synthetic_keypoints_rect_smoke | 24 | 2304 | 96x192 images, 21x21 patches | rectangular-image robustness |
| synthetic_keypoints_small_blur | 64 | 6144 | 48x48 images, 15x15 patches, strong blur | noise/blur robustness |
| synthetic_keypoints | 200 | 19200 | 128x128 images, 21x21 patches | main training/evaluation dataset |

Each `.npz` label file contains `image`, `points_xy`, `type_ids`, `heatmap`, `scene_type`, `patch_centers_xy`, `patch_labels`, and `patches`. Patch labels are balanced in the supplied datasets, but still compute `pos_weight` from the training split to keep the code robust.

## 6. AGENTS.md to add at repository root

```markdown
# AGENTS.md

## Project goal
Build a reproducible research-code pipeline for QPP-inspired / data-reuploading QNN corner and keypoint detection. The task is patch-level binary classification: local image patch or candidate point -> corner/junction/keypoint probability.

## Working rules
- Use PyTorch exact statevector simulation for 1-qubit and 2-qubit QNNs. Do not require PennyLane, Qiskit, or quantum hardware for the default path.
- Keep OpenCV optional. If it is unavailable, skip FAST/Harris image-level baselines gracefully and keep pure NumPy/SciPy baselines working.
- Avoid data leakage. Split by image/sample id, fit normalizers only on train, and choose thresholds only on validation.
- Save all generated artifacts under outputs/ and never modify raw data.
- Every script must have CLI help and reproducible seed control.
- After code changes, run pytest and at least one smoke experiment when feasible.
```

## 7. Codex prompt

Paste the following into Codex from the repository root.

```text
You are a senior research-code engineer. Build a reproducible Python project for a QPP-inspired / data-reuploading QNN corner/keypoint detection experiment.

Repository context:
- The project root contains `data/raw/data.zip`, an uploaded dataset archive of about 35 MB.
- When extracted with `unzip data/raw/data.zip -d data/raw`, it creates `data/raw/data/` containing subdatasets: `synthetic_keypoints`, `synthetic_keypoints_small_blur`, `synthetic_keypoints_smoke`, `synthetic_keypoints_rect_smoke`, `synthetic_keypoints_square_smoke`, and `smoke_readme_pipeline`.
- Each subdataset has `config.json`, `manifest.jsonl`, `images/*.png`, and `labels/*.npz`.
- Each `.npz` contains at least `image`, `points_xy`, `type_ids`, `heatmap`, `scene_type`, `patch_centers_xy`, `patch_labels`, and `patches`.

Goal:
Implement a complete experiment pipeline for patch-level binary classification. Given a local patch or candidate center point, predict whether it is a corner / junction / keypoint. Treat all positive keypoint categories as one positive class.

Core modeling direction:
1. Main model A: 2-qubit QNN with two structure-tensor features `[lambda1, lambda2]`.
2. Main model B: 2-qubit QNN with two cornerness features `[logS, eta]`, where `S=lambda1+lambda2`, `logS=log(S+eps)`, and `eta=4*lambda1*lambda2/(S^2+eps)` clipped to `[0,1]` up to numerical tolerance.
3. Main model C: 1-qubit scalar QPP-inspired QNN with scalar `t`. Support `t=logS+c*eta` with `c` swept over `[0.0, 0.25, 0.5, 1.0, 2.0, 4.0]`; also support `t=lambda2`, `t=R`, and an optional learnable linear projection from `[logS, eta]` to `t`.
4. Include `[Ix, Iy]` as a deliberately weak ablation input.
5. Include 5D `[Ix, Iy, lambda1, lambda2, R]` as a reference feature set.

Required repository structure and files:
- `src/qpp_corner/data.py`: dataset discovery, manifest reading, `.npz` loading, and group-aware image-level splitting. Resolve manifest paths robustly whether the data root is `data/raw` or `data/raw/data`.
- `src/qpp_corner/features.py`: finite-difference or Sobel gradients, Gaussian-weighted structure tensor, sorted eigenvalues, Harris response, Hessian features, feature-set construction.
- `src/qpp_corner/normalizer.py`: train-only feature normalization, clipping to `[-3,3]`, angle mapping `phi = pi*z/3`, save/load JSON.
- `src/qpp_corner/qnn_torch.py`: exact differentiable 1q and 2q data-reuploading QNNs using complex PyTorch tensors. Implement `Ry`, `Rz`, single-qubit gate application, CNOT, expectation values `<Z0>`, `<Z1>`, `<Z0Z1>`, and trainable affine readout. Use logits and `BCEWithLogitsLoss`.
- `src/qpp_corner/classical.py`: logistic regression, small MLP, validation-selected threshold baselines on `R`, `lambda2`, `S`, `eta`, and `logS+eta`; optional OpenCV Harris/FAST baselines when OpenCV is installed.
- `src/qpp_corner/metrics.py`: accuracy, precision, recall, F1, ROC-AUC, PR-AUC; choose classification threshold on validation by best F1 and apply it to test.
- `src/qpp_corner/train.py`: shared train/eval loops, early stopping, checkpointing, history logging.
- `src/qpp_corner/viz.py`: PR curves, training curves, feature scatter plots, and image/heatmap/predicted-keypoint previews.
- `scripts/inspect_data.py`: print dataset counts, scene-type counts, patch shapes, label balance, and npz keys.
- `scripts/build_features.py`: build and cache feature matrices under `data/processed/`.
- `scripts/run_experiment.py`: run one YAML config and save metrics/checkpoint/normalizer.
- `scripts/run_ablation.py`: run the planned ablation matrix and write `outputs/runs/<run_id>/summary.csv`.
- `scripts/eval_keypoints.py`: optional second-stage evaluation: reconstruct sparse score maps from patch centers, apply threshold + NMS, compare detections to `points_xy` within a configurable radius.
- `configs/*.yaml`: create smoke and full configs for 2q-lambda12, 2q-logS-eta, 1q-scalar, and ablation suites.
- `tests/`: lightweight tests for data loading, feature extraction, normalizer, QNN forward pass, and metric thresholding.
- `README.md`: exact commands for install, data extraction, inspection, smoke run, full run, ablation run, and preview generation.

Feature extraction details:
- For each patch, compute gradients after optional Gaussian smoothing.
- Center-gradient features: `Ix_center`, `Iy_center` at the central pixel.
- Structure tensor over the whole patch: `M=[[sum(w*Ix^2), sum(w*Ix*Iy)], [sum(w*Ix*Iy), sum(w*Iy^2)]]`, using Gaussian weights centered in the patch.
- Sort eigenvalues descending: `lambda1 >= lambda2 >= 0`.
- `R = det(M) - k * trace(M)^2`, default `k=0.04`.
- `S = trace(M) = lambda1 + lambda2`.
- `eta = 4*det(M)/(trace(M)^2 + eps)`.
- `logS = log(S + eps)`.
- Add Hessian features `[Ixx, Ixy, Iyy, detH, trH]` for later ablations, but do not make them required for the first smoke run.

QNN details:
- 2q-QNN inputs are two angle features `(phi1, phi2)`.
- Each layer applies encoding -> trainable single-qubit rotations -> entanglement.
- Encoding options: `ry`, `ryrz`, and `qpp_z` if useful.
- Suggested 2q layer: `Ry(phi0) Rz(phi0)` on q0 and `Ry(phi1) Rz(phi1)` on q1; then trainable `Rz Ry Rz` on each qubit; then CNOT entanglement. Support `none`, `linear_01`, and `bidirectional` entanglement.
- 2q readout: `[<Z0>, <Z1>, <Z0Z1>]` followed by a trainable linear readout to one logit.
- 1q scalar QNN: repeat `Rz(t)` or `Ry(t)Rz(t)` data reuploading with trainable `Rz Ry Rz`; readout `<Z>` plus trainable affine head.
- Hyperparameters: `L in [1,2,3,4,6]`, Adam, LR grid `[1e-3, 3e-3, 1e-2]`, batch size 64 or 128, epochs 50-200, early stopping on validation PR-AUC or F1.

Data protocol:
- Use image-level/group-level splits, not random patch-level splits, to avoid leakage. Groups are sample ids from the manifest.
- First run smoke tests on `smoke_readme_pipeline` or `synthetic_keypoints_smoke`.
- Then train on `synthetic_keypoints` with 70/15/15 train/val/test image-level split.
- Include robustness tests: train on `synthetic_keypoints`, test on `synthetic_keypoints_small_blur`, `synthetic_keypoints_rect_smoke`, `synthetic_keypoints_square_smoke`, and smoke variants.

Baselines and comparisons:
- Logistic regression on the exact same feature set.
- Small classical MLP on the exact same feature set.
- Threshold baselines using validation-selected thresholds on `R`, `lambda2`, `S`, and `logS+eta`.
- Optional OpenCV Harris/FAST for image-level/keypoint-level evaluation.
- QNN ablations: feature set, depth L, encoding type, entanglement type, readout, 1q scalar definition, and c sweep.

Outputs:
- Save every run under `outputs/runs/<timestamp_or_run_id>/`.
- Save `config_resolved.yaml`, `metrics.json`, `history.csv`, `normalizer.json`, `best_model.pt`, plots, and prediction previews.
- The ablation runner must produce a compact CSV with columns: dataset, split, feature_set, model, n_qubits, L, encoding, entanglement, readout, val_pr_auc, val_f1, test_pr_auc, test_f1, test_roc_auc, threshold, notes.

Quality requirements:
- Reproducible seeds.
- Clear CLI help for all scripts.
- No data leakage: normalizer fitted only on train split; threshold selected only on validation split.
- If optional dependency is missing, skip optional baseline gracefully.
- Use small defaults so smoke runs finish quickly.

After implementing, run at least:
1. `python scripts/inspect_data.py --data-root data/raw`
2. `python scripts/run_experiment.py --config configs/smoke_2q_logS_eta.yaml`
3. `python scripts/run_experiment.py --config configs/smoke_2q_lambda12.yaml`
4. `python scripts/run_experiment.py --config configs/smoke_1q_scalar.yaml`
5. `python scripts/run_ablation.py --config configs/ablation_smoke.yaml`
6. `pytest -q`

Report exactly what succeeded, what failed, the key smoke metrics, and the exact next commands for the full run.
```
